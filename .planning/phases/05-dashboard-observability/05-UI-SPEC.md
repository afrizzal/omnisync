## Spacing Scale

Declared values (all multiples of 4 ‚Äî 8-point grid via Tailwind v4 utility classes):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon gaps, inline badge padding |
| sm | 8px | `gap-2` / `p-2` | Compact element spacing, table cell padding |
| md | 16px | `gap-4` / `p-4` | Default element spacing, card padding |
| lg | 24px | `gap-6` / `p-6` | Section padding, card header |
| xl | 32px | `gap-8` / `p-8` | Layout gaps between page sections |
| 2xl | 48px | `gap-12` | Major section breaks (between metrics grid and chart) |
| 3xl | 64px | `gap-16` | Page-level top/bottom padding |

**Exceptions:**
- Metrics stat number: `text-4xl` (36px) for the count value inside Card ‚Äî oversized for impact at a glance
- DLQ table: `max-w-xs truncate` on failure reason cell to prevent overflow
- Chart container: `height={400}` fixed (Recharts `ResponsiveContainer width="100%"`)

**Source:** Default 8-point scale (standard); exceptions from RESEARCH.md Pattern 6 (chart height) and Pattern code examples (truncate).

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 | Table cell content, badge labels, error messages |
| Label | 16px (`text-base`) | 400 (regular) | 1.5 | Card titles, form labels, nav links |
| Heading | 20px (`text-xl`) | 600 (semibold) | 1.2 | Page headings (h1 equivalent per page) |
| Display | 36px (`text-4xl`) | 600 (semibold) | 1.0 | Metric stat numbers (queue count, throughput) |

**Weights declared:** regular (400) + semibold (600) ‚Äî exactly 2 weights.

**Note:** shadcn/ui `CardTitle` uses `font-semibold` by default (600). Metric stat numbers inside `CardContent` also use `font-semibold` (600). The `text-4xl` size (36px) provides sufficient visual hierarchy for at-a-glance count scanning without a third weight. Use `className="text-4xl font-semibold"` for Display role elements.

**Source:** shadcn/ui default component styles (CardTitle = semibold); RESEARCH.md Pattern code examples; standard body/label convention.

## Color

**Theme:** System-aware dark/light via `next-themes`. All color values are shadcn CSS variables ‚Äî they resolve differently in light and dark mode. The hex values below are the light-mode equivalents for reference.

| Role | CSS Variable | Light-Mode Approx | Usage |
|------|-------------|-------------------|-------|
| Dominant (60%) | `hsl(var(--background))` | #ffffff | Page background, main content area |
| Secondary (30%) | `hsl(var(--card))` | #f9fafb | Card surfaces, table background, nav sidebar |
| Accent (10%) | `hsl(var(--primary))` | #18181b | Primary interactive elements (see reserved list below) |
| Destructive | `hsl(var(--destructive))` | #ef4444 | Destructive-variant Badges, error states, failed series on chart |

**Accent reserved for (explicit list):**
- "Start Load Test" button on `/demo` page ‚Äî the primary CTA in Phase 5
- Active nav link indicator (current page highlight)
- Chart `completed` series stroke and fill: `stroke="#22c55e"` (Tailwind green-500) ‚Äî this is a semantic data color, not the shadcn accent token

**Chart semantic colors (outside shadcn token system):**
- `completed` series: `stroke="#22c55e"` fill `"#bbf7d0"` (green-500 / green-100)
- `failed` series: `stroke="#ef4444"` fill `"#fecaca"` (red-500 / red-100) ‚Äî matches `--destructive`

**Badge variants:**
- `variant="destructive"` ‚Äî FAILED status, unresolved DLQ count, attempt count
- `variant="secondary"` ‚Äî RESOLVED status (when a DLQ entry has been re-queued successfully)
- `variant="outline"` ‚Äî neutral status labels

**Source:** CONTEXT.md D-13, D-14; RESEARCH.md Pattern 6 (chart color literals); shadcn/ui CSS variable system.

## Interaction Contract

### Polling behavior

- **Interval:** 3000ms default (env: `DASHBOARD_POLL_INTERVAL_MS`). Configurable to 1000ms for demos.
- **First fetch:** Immediate on mount (no initial 3s delay).
- **Cleanup:** `clearInterval` on unmount. Required ‚Äî missing cleanup causes doubled polls on route change.
- **While loading:** Render `"‚Äî"` for all numeric values.
- **On error:** Show `"Unable to reach API. Retrying..."` banner. Continue polling (do not stop interval on error).
- **Recovery:** Auto-recovers when API becomes reachable ‚Äî no manual refresh required.

### Re-queue button

- **State machine:** idle ‚Üí loading ‚Üí success | error
- **Loading state:** Button disabled + spinner (or just `disabled` + "Re-queuing..." label).
- **Success:** Update row badge to `variant="secondary"` label "RESOLVED" + show toast "Re-queued successfully."
- **Already queued:** Toast "Already queued ‚Äî this event is being processed." Row stays FAILED.
- **Not found:** Toast "Entry not found. It may have been resolved already." Row stays visible.
- **No confirmation dialog.** Re-queue is idempotent ‚Äî single click sufficient.

### Theme toggle

- Toggle button in NavBar: cycles system ‚Üí light ‚Üí dark.
- Uses `next-themes` `useTheme()` hook.
- Icon: text labels ("Light" / "Dark" / "System") or a simple sun/moon SVG inline ‚Äî no icon library needed.

### Chart accumulation

- Each poll appends one data point `{ timestamp: string, completed: number, failed: number }`.
- Rolling window: last 60 points (60 √ó 3s = 3 min of history).
- No animation between points (remove `isAnimationActive` default or set to `false` for live data ‚Äî prevents jitter).
- X-axis: `toLocaleTimeString()` formatted timestamp.
- Y-axis: auto-scaled by Recharts.
- Tooltip: default Recharts tooltip (shows both series values on hover).

**Source:** CONTEXT.md D-01, D-02, D-03, D-11, D-16; RESEARCH.md Pattern 5, Pattern 6.
