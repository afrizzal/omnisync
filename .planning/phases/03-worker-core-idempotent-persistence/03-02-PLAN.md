<objective>
Harden the shared packages so the worker can be built safely in Wave 1 and the producer↔consumer contract is explicit. Refactor `@omnisync/queue` to side-effect-free factories (D-07/D-08), remove the dead `guardInterval` config (D-09 amendment), add the `EventJobData` schema codifying the wire contract (D-10), add `WORKER_CONCURRENCY` to config (D-12), and rewire `apps/api/src/index.ts` onto the factories while dropping its `@omnisync/db` dependency (D-14). All 25 existing API tests must stay green.

Purpose: Importing a constant must never open a socket — the current `packages/queue` opens a Redis connection at import time, which makes the package untestable and dangerous for the worker. The `guardInterval` config has zero effect in BullMQ v5 (QueueScheduler removed in v2) and must go. Making `EventJobData` explicit lets the worker `safeParse` job data as a poison-message guard. Dropping `@omnisync/db` from the API turns "API never touches the DB" from a convention into a structural property.
Output: Side-effect-free queue factories with a test proving import opens no socket, the formalized job contract, the new env var, and a rewired API entrypoint — all with the existing API suite green.
</objective>

```typescript
import { env } from "@omnisync/config";
import { createEventsQueue, createRedisConnection } from "@omnisync/queue";
import { buildApp } from "./app.js";

const connection = createRedisConnection(env.REDIS_URL);
const queue = createEventsQueue(connection);

const port = Number.parseInt(process.env.PORT ?? "3001", 10);
const host = process.env.HOST ?? "0.0.0.0";

  await connection.quit();
  process.exit(0);
}
process.on("SIGINT", () => void shutdown());
process.on("SIGTERM", () => void shutdown());
```
