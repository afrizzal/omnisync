The REAL ingest path (apps/api/src/routes/ingest.ts) does, in order:
  1. HMAC verify via verifySignature(rawBody, secret, header)  — header MUST start with "sha256="
  2. Zod validate { ...body, source: source.toUpperCase() } against InboundEvent
  3. fingerprint → Redis SET NX dedup gate → BullMQ enqueue (jobId = fingerprint)

verifySignature (apps/api/src/lib/hmac.ts):
  - Returns false unless signatureHeader.startsWith("sha256=").
  - expected = createHmac("sha256", secret).update(rawBody).digest("hex"); compares the hex after the prefix.

getSecretForSource (apps/api/src/lib/secrets.ts):
  export function getSecretForSource(source: string): string | null
  - Keyed by UPPERCASE source: SHOPEE | TOKOPEDIA | META_ADS | CRM. Returns null for unknown.

<task type="auto">
  <name>Task 2: Fix B (OPS-04) — wire POST /api/demo/start to fire a background burst of signed synthetic events</name>
  <files>apps/api/src/routes/demo.ts</files>
  <action>
Rewrite apps/api/src/routes/demo.ts so `POST /api/demo/start` drives the REAL ingest pipeline via `app.inject` (self-contained — exercises HMAC + Zod + dedup + enqueue). Keep the export signature `export async function demoRoutes(app: FastifyInstance): Promise<void>` (app.ts calls `await demoRoutes(instance)`; app.inject is on the Fastify instance).

Imports: `import { createHmac } from "node:crypto";` and `import { getSecretForSource } from "../lib/secrets.js";` (note the `.js` extension — NodeNext ESM).

Handler requirements:
  - Return `reply.code(202).send({ status: "started" })` IMMEDIATELY. Fire events in the BACKGROUND — do NOT await the whole burst before replying.
  - Burst shape: ~12 seconds total, ~20 events/sec (~240 events) via a recursive setTimeout (50ms interval) so the chart visibly climbs across multiple 3s metrics polls. A simple round-robin counter selects the source each tick.
  - Spread across all 4 sources, LOWERCASE in the URL: ["shopee", "tokopedia", "meta_ads", "crm"] → POST `/ingest/<source>`.
  - For each event:
      const source = sources[i % 4];
      const body = JSON.stringify({
        eventType: "order.created",
        externalId: `demo-${Date.now()}-${Math.random().toString(36).slice(2)}`,
        occurredAt: new Date().toISOString(),
        payload: { amount: Math.floor(Math.random() * 1000) },
      });
      const secret = getSecretForSource(source.toUpperCase());
      if (!secret) continue; // skip unconfigured source rather than 401-spam
      const signature = "sha256=" + createHmac("sha256", secret).update(body).digest("hex"); // CRITICAL prefix
      await app.inject({
        method: "POST",
        url: `/ingest/${source}`,
        payload: body,                     // pass the EXACT same string as the HMAC input
        headers: { "content-type": "application/json", "x-webhook-signature": signature },
      });
    Pass the EXACT same `body` string to BOTH the HMAC input and the inject payload so rawBody bytes match the signature. (Do NOT include `source` in the body — the /ingest route injects it.)
  - Wrap each inject in try/catch so one bad inject never kills the burst loop (log + continue).
  - Guard against overlapping concurrent runs: a module-level `let running = false` boolean. If already running, still return 202 (so the button is idempotent) but skip starting a second burst. Reset `running = false` when the burst finishes.
  - Nice-to-have: one structured `app.log.info(...)` line on start (e.g. count + duration).

Build/typecheck must pass: `pnpm --filter @omnisync/api build` (TypeScript strict + NodeNext). Avoid floating-promise lint issues — kick off the background burst from a named function and intentionally not await it (e.g. `void runBurst();`).
