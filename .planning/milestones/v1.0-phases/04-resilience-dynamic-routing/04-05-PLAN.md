# Plan 04-05: Re-queue API + docker-compose mock-crm service

## Objective

Build the operator re-queue path (RES-06): a service-layer function + `POST /admin/dlq/:id/requeue` endpoint on `apps/api` that re-enqueues a Dead-Letter entry through the normal worker pipeline, idempotently. Per key research finding #6, re-queue uses `queue.add(name, data, { jobId: fingerprint })` (NOT `job.retry()`) — reusing the fingerprint as jobId means BullMQ deduplication + the DB `ON CONFLICT (fingerprint)` constraint guarantee exactly one row even if re-queued twice. Also add the `mock-crm` docker-compose service so the worker's `HttpCrmClient` can reach it (D-06/D-09).

**Decision (Claude's discretion, D-16):** re-queue lives on a dedicated `/admin` router on `apps/api`. The dashboard (Phase 5) calls this endpoint — it does not own the business logic. Bulk re-queue is deferred to Phase 5 (this plan ships single-entry only).

**Architectural note:** `apps/api` must read `dlq_events` to fetch the entry to re-queue. Phase 3 D-14 made the API structurally DB-free for the ingestion path. Re-queue is an admin operation, not the hot ingestion path — it legitimately needs DB read access. Add `@omnisync/db` back as a dependency of `apps/api` SCOPED to the admin router only; the ingest hot path remains DB-free (ING-05 stays an architectural property of the ingest route, enforced by it never importing prisma).

Step 2 — create `apps/api/src/services/requeue.ts`. The service is DI-friendly (mirrors buildApp): it takes a prisma-like and a queue-like dependency so tests can inject fakes.

```ts
import type { PrismaClient } from "@omnisync/db";
import type { Queue } from "bullmq";

export interface RequeueDeps {
  prisma: Pick<PrismaClient, "deadLetterEvent">;
  queue: Pick<Queue, "add">;
}

export type RequeueResult =
  | { status: "requeued"; fingerprint: string }
  | { status: "not_found" }
  | { status: "already_queued"; fingerprint: string };

// RES-06: re-enqueue a DLQ entry through the NORMAL worker pipeline. Idempotent by construction:
// jobId = fingerprint -> BullMQ deduplicates an in-flight/duplicate re-queue; the worker's
// ON CONFLICT (fingerprint) DO NOTHING absorbs any duplicate at the DB (IDM-02/IDM-03).
// NOTE: we use queue.add (NOT job.retry) per research finding #6 — the failed BullMQ job may have
// been removed (removeOnFail age), but the Postgres dlq_events mirror is the durable source of truth.
export async function requeueDlqEntry(
  deps: RequeueDeps,
  id: string,
): Promise<RequeueResult> {
  const entry = await deps.prisma.deadLetterEvent.findUnique({ where: { id } });
  if (!entry) return { status: "not_found" };

  // Reconstruct EventJobData from the durable DLQ row. payload is the stored InboundEvent JSON.
  const jobData = {
    source: entry.source,
    payload: entry.payload,
    fingerprint: entry.fingerprint,
  };

  const job = await deps.queue.add("process-event", jobData, {
    jobId: entry.fingerprint, // reuse fingerprint -> BullMQ dedup makes re-queue idempotent
  });

  // BullMQ returns the existing job (no new add) when the jobId already exists in active/waiting;
  // treat a re-add of a still-present job as already_queued (Pitfall 8 double-click safety).
  if (job == null) return { status: "already_queued", fingerprint: entry.fingerprint };

  // Mark the DLQ entry resolved so the dashboard reflects the re-queue (optional but cheap).
  await deps.prisma.deadLetterEvent.update({
    where: { id },
    data: { resolved: true },
  });

  return { status: "requeued", fingerprint: entry.fingerprint };
}
```
(If the prisma type narrowing on `Pick<PrismaClient, "deadLetterEvent">` does not expose `findUnique`/`update` cleanly, widen `prisma` to `PrismaClient` — the DI fake in tests can still be a partial cast. Prefer the full `PrismaClient` type if narrowing is awkward.)

Step 3 — create `apps/api/src/routes/admin.ts`:
```ts
import type { FastifyInstance } from "fastify";
import { requeueDlqEntry, type RequeueDeps } from "../services/requeue.js";

export async function adminRoutes(
  app: FastifyInstance,
  deps: RequeueDeps,
): Promise<void> {
  app.post("/admin/dlq/:id/requeue", async (request, reply) => {
    const { id } = request.params as { id: string };
    const result = await requeueDlqEntry(deps, id);

    if (result.status === "not_found") {
      return reply.code(404).send({
        error: "DLQ_ENTRY_NOT_FOUND",
        message: `No DLQ entry with id ${id}`,
      });
    }
    if (result.status === "already_queued") {
      return reply.code(200).send({
        status: "already_queued",
        fingerprint: result.fingerprint,
      });
    }
    return reply.code(200).send({
      status: "requeued",
      fingerprint: result.fingerprint,
    });
  });
}
```
