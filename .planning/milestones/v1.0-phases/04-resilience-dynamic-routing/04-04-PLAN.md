# Plan 04-04: Wire resilience into the worker

## Objective

Integrate the Wave 0 primitives into the live worker pipeline. After this plan: retries use full-jitter backoff (RES-01), exhausted jobs mirror to Postgres DLQ (RES-02/RES-03), the CRM sync is circuit-breaker-guarded after a successful persist (RES-04/RES-05), Postgres failures bypass the CRM breaker and propagate to BullMQ retry (RES-07), and `normalize()` applies cached routing rules (RTE-01/RTE-02). The cockatiel policy is created ONCE in `index.ts` and injected as a singleton via `WorkerDeps` (Pitfall 1).

This plan EXTENDS existing files — it must not break the 4 verified Phase 3 worker behaviors or their passing unit tests.

## must_haves
- [ ] `apps/worker/src/worker.ts` `WorkerDeps` includes `crmClient: CrmClient` and `crmPolicy: CircuitBreakerPolicy`; Worker registers `settings.backoffStrategy: fullJitterBackoff`; `worker.on("failed")` calls the DLQ handler
- [ ] `apps/worker/src/processor/event.processor.ts` `buildProcessor` accepts `crmClient` + `crmPolicy` and calls `crmPolicy.execute(() => crmClient.sync(normalized))` AFTER `persistEvent`
- [ ] `apps/worker/src/normalizer/normalize.ts` `normalize` applies routing rules at the seam (signature change documented below)
- [ ] `apps/worker/src/index.ts` creates `createCrmPolicy(env.BREAKER_HALF_OPEN_MS)` + `new HttpCrmClient(env.CRM_BASE_URL)` once and injects them
- [ ] `processor.test.ts` extended: proves CRM sync is called; proves Postgres failure does NOT trip the breaker (RES-07)
- [ ] `pnpm --filter @omnisync/worker test` exits 0 (all unit tests, including the 3 prior Phase 3 worker tests, stay green)

```ts
import type { CircuitBreakerPolicy } from "cockatiel";
import type { CrmClient } from "../crm/crm-client.js";
import { getActiveRules } from "../normalizer/rule-cache.js";

export function buildProcessor(
  prisma: PrismaClient,
  logger: ProcessorLogger,
  crmClient: CrmClient,
  crmPolicy: CircuitBreakerPolicy,
  ttlMs: number,
) {
  return async function processEvent(job: Pick<Job, "id" | "data">): Promise<void> {
    // ... existing poison guard unchanged ...
    const parsed = EventJobData.safeParse(job.data);
    if (!parsed.success) { /* unchanged throw */ }

    // RTE-02: load active rules via lazy TTL cache (DB hit only when TTL expired)
    const rules = await getActiveRules(prisma, ttlMs);
    const normalized = normalize(parsed.data, rules);

    // Persist FIRST. Postgres failure throws here and propagates straight to BullMQ retry —
    // it must NOT touch the CRM breaker (Pitfall 4 / RES-07). persistEvent is NOT inside execute().
    const outcome = await persistEvent(prisma, normalized);

    // RES-04/RES-05: CRM sync AFTER successful persist, guarded by the cockatiel breaker.
    // BrokenCircuitError (open breaker) and CRM errors both throw -> BullMQ retries with backoff.
    await crmPolicy.execute(() => crmClient.sync(normalized));

    // ... existing duplicate/completed logging unchanged ...
  };
}
```

CRITICAL ordering (Pitfall 4 / RES-07): `persistEvent(prisma, ...)` is called OUTSIDE `crmPolicy.execute()`. Only `crmClient.sync(normalized)` goes inside the breaker. A Postgres outage makes `persistEvent` throw before the breaker is ever consulted, so the CRM breaker stays closed — proving RES-07's "killing Postgres does NOT open the breaker".
